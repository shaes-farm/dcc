# ADR-0001: Trace viewer joins the §5.3 panel library

**Date:** 2026-07-22
**Status:** accepted
**Spec:** §5.3, §3.2
**Issue:** [#4](https://github.com/shaes-farm/dcc/issues/4)

## Context

§3.2 gives every canonical object a URI, and `trace://tempo/4bf92f35` is one of
them. §5.3's v1 panel library — Workspace Health, Repos grid, PRs, Workflow
Runs, Security, Environments, Pods, Pod detail, Deploys, Logs, Log search, REST
explorer, GraphQL explorer, Health board, Error rates, Latency, Pinned
dashboard, Context, Documents, Document viewer, Lineage strip, Artifacts, Audit
log — has no trace panel.

The gap was invisible until the `Uri → { panel, params }` resolver
([#4](https://github.com/shaes-farm/dcc/issues/4)) made it a compile-time
question: the resolver switches exhaustively over `URI_SCHEMES`, so every scheme
has to name a destination. `trace` had none.

Nothing else in the spec suggests this was deliberate. §2.2 already declares
`getTrace?(traceId: string): Promise<Trace>` on `ObservabilityProvider`, §3.1
lists `Trace` as a canonical object, and §6.4 says log lines link to traces when
`trace_id` is present. A URI scheme, a provider method, a canonical object, and
an inbound link — with nothing to render at the end of them.

## Decision

The v1 panel library gains **Trace viewer**, panel id `trace-viewer`.
`trace://<provider>/<traceId>` resolves to it.

The id ships with the resolver now. The panel itself is Phase 4 (§11,
Observability), alongside the rest of §6.4.

`docs/ARCHITECTURE.md` is **not** edited — it stays frozen at v0.4. §5.3 is read
together with this ADR.

## Consequences

- `PANEL_IDS` carries an id with no component behind it until Phase 4. That is
  visible in one place and obviously provisional, which beats a scheme that
  parses fine and then throws when someone pastes a trace URI into the palette.
- Whoever builds §6.4 inherits a decision instead of making one, and the
  correlation thread (§6.4, Phase 5) already has a URI to point its trace row at.
- §5.3 can no longer be read alone. That cost is the general cost of freezing a
  spec and recording amendments elsewhere, and it is why [index.md](index.md)
  exists.

## Alternatives considered

**Leave `trace://` unresolvable until Phase 4.** It would have to fail somewhere
— a runtime throw, or a scheme quietly missing from the resolver's switch. Both
break "everything is addressable" (§3.2) for one scheme, and the failure surfaces
at the worst moment: an engineer pasting a trace link during an incident.

**Map `trace://` onto the correlation thread.** A trace is a leaf object; the
correlation thread is a Phase 5 narrative stitched across deploy, metric, log,
trace, and PR (§6.4). Pointing a leaf URI at the story it happens to appear in
conflates two objects, and would leave Phase 4 with no way to open a trace on its
own.

**Amend `docs/ARCHITECTURE.md` and bump it to v0.4.1.** Cheapest diff, but it
reopens a document deliberately frozen for a reason — the spec growing more
sophisticated than the software is called out in §11 as the project's biggest
risk. Recording the decision beside the spec keeps the freeze real and makes
every post-freeze change reviewable on its own.
